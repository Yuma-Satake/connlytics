# 機能ロードマップ

## Phase 1: Markdownプレビュー機能

### 概要

イベント説明文のMarkdownをリアルタイムでプレビュー・編集できる基本機能を実装。

### 機能一覧

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| F-WEB-001 | Markdownエディタ | シンタックスハイライト付きエディタ | 必須 |
| F-WEB-002 | Markdownプレビュー | connpassスタイルでのリアルタイムプレビュー | 必須 |
| F-WEB-003 | Chrome拡張連携 | 拡張機能からのデータ受信 | 必須 |
| F-WEB-004 | エクスポート機能 | コピー・ダウンロード機能 | 必須 |
| F-EXT-001 | connpassページ検知 | イベントページの自動検知 | 必須 |
| F-EXT-002 | Markdown抽出 | イベント説明のMD抽出 | 必須 |
| F-EXT-003 | Webアプリ連携 | MDデータの送信 | 必須 |
| F-EXT-004 | Popup UI | 拡張機能のUI | 必須 |

### 成果物

- Webアプリケーション（Next.js）
- Chrome拡張機能（Manifest V3）

## Phase 2: 参加者分析機能（将来実装）

### 概要

イベント参加者のデータを分析し、継続参加者の特定や参加率の可視化を行う。

### 想定機能

| ID | 機能 | 説明 |
|----|------|------|
| F-WEB-101 | 参加者リスト取得 | connpassから参加者データを取得 |
| F-WEB-102 | 継続参加者分析 | 複数イベントに参加しているユーザーを特定 |
| F-WEB-103 | 継続率ダッシュボード | 継続率をグラフで可視化 |
| F-WEB-104 | 参加者セグメント | 参加頻度でユーザーを分類 |

### 検討事項

- connpassのAPIまたはスクレイピングによるデータ取得方法
- データの保存場所（ローカル or クラウド）
- プライバシーへの配慮
